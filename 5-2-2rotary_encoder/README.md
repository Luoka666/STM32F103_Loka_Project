# 旋转编码器计次实验 (EXTI 外部中断)

## 项目简介
本项目通过 STM32 的 EXTI 外部中断功能，实现了对旋转编码器（正交信号）的计次。利用 A、B 两相的相位差判断转动方向，并实时更新计数器值。

## 核心排错日志 

### 1. 故障现象：计数器数值不改变
**问题描述**：
旋转编码器时，OLED 显示的数值保持不变，或者在 `++` 和 `--` 之间瞬间跳变后回到原值（例如：从 10 变成 11 后立刻变回 10）。

**根因分析**：
* **单边思维陷阱**：最初的逻辑仅仅捕获了 A、B 引脚的下降沿。由于编码器顺时针转动时，A 相产生下降沿，B 相随后也会产生下降沿。
* **逻辑冲突**：原代码在 `EXTI0` 触发时执行 `counter++`，在 `EXTI1` 触发时执行 `counter--`。结果转动一格时，两个中断先后触发，导致加减抵消。
* **错误代码如下**：

```c
	void EXTI0_IRQHandler(void) {
	  if (EXTI_GetITStatus(EXTI_Line0) != RESET) {
	      if (GPIO_ReadInputDataBit(GPIOB,GPIO_Pin_0) == 0) {
	          counter++;
	      }
	  }
    EXTI_ClearITPendingBit(EXTI_Line0);
	}
```

**解决方案（正交解码逻辑）**：

必须利用两相信号的**相位差**。在其中一相发生跳变的瞬间，去读取另一相的电平状态。
* **顺时针**：A 下降沿瞬间，B 尚未跳变，处于**高电平**。
* **逆时针**：A 下降沿瞬间，B 已经跳完，处于**低电平**。

**解决方案代码实现**：

```c
void EXTI0_IRQHandler(void) {



    if (EXTI_GetITStatus(EXTI_Line0) != RESET) {
        //A相跳变（但还不知道A相是在B相之前还是之后跳变）
        if (GPIO_ReadInputDataBit(GPIOB,GPIO_Pin_1) == 0) {//判断B相状态，如果B相为0，则说明A相后跳变，逆时针，counter--

            counter--;

        }else {//判断B相状态，如果B相为1，则说明A相先跳变，顺时针，counter++

            counter++;

        }

    }

    EXTI_ClearITPendingBit(EXTI_Line0);

}
```

### 2. 故障现象：数值抖动，旋转一次计数多次
**问题描述**：
转动一格编码器，计数值跳变了 10 几个甚至更多。

**原因分析**：
* **机械弹跳**：旋转编码器内部是金属片摩擦，在信号跳变的瞬间会产生极高频的杂波（抖动）。
* **触发过敏**：EXTI 速度极快，会把每一次抖动都当成一次真实的转动信号，导致多次计数。

**后续优化方向（还未实现）**：
1. **硬件消抖**：引脚并联 0.1μF 电容。
2. **软件方案**：下一阶段尝试使用定时器（TIM）的 **Encoder Interface** 模式，利用硬件滤波机制彻底解决。
